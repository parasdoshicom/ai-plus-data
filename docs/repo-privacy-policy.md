# Repo Privacy Policy

This repository is public. That creates a simple rule:

If a piece of information would not be safe to paste into a public conference talk, it does not belong here.

## Allowed

- generalized lessons
- rewritten workflow patterns
- synthetic examples
- public links and public writing
- role-level guidance for analysts, managers, and data leaders

## Not Allowed

- private company operating documents
- internal links, IDs, dashboard references, or system wiring
- customer-level or employee-level data
- internal metrics that are not already public
- compensation, performance, HR, or personnel details
- secrets, tokens, credentials, or auth instructions
- private Telegram, Slack, email, or bot runtime configuration

## Rewrite Standard

Private source material can inform a public artifact only when all of the following are true:

1. the result is meaningfully rewritten
2. the result is generalized beyond one company
3. sensitive names, metrics, links, and identifiers are removed
4. the final artifact stands on its own as public educational material

## Example Policy

Acceptable:

- "Use AI to generate a first-pass weekly KPI briefing with decomposition and validation."

Not acceptable:

- copying a private briefing template with real company metrics, table names, or links

## Operating Rule

When in doubt, exclude the detail and keep the public lesson.

# Publication Safety Checklist

Run this before committing or publishing new material to this repo.

## Secrets and Credentials

- [ ] No API keys, tokens, secrets, passwords, or credential examples copied from real systems
- [ ] No auth instructions tied to private infrastructure
- [ ] No environment files or runtime config from private workspaces

## PII and Sensitive Data

- [ ] No customer names, emails, phones, addresses, or IDs
- [ ] No employee or candidate personal information
- [ ] No UUIDs, tokens, chat IDs, or other operational identifiers

## Company Confidentiality

- [ ] No internal URLs, dashboard links, doc IDs, or Slack links
- [ ] No private metrics, SQL, schemas, or table catalogs
- [ ] No compensation, performance, or HR material
- [ ] No private meeting notes or strategy docs

## Wrong-Company Leakage

- [ ] No immigration or unrelated portfolio-company language
- [ ] No copied Telegram wiring or bot instructions
- [ ] No private operating terminology that assumes access to another workspace

## Example Quality

- [ ] Every example is synthetic, toy, or explicitly public-safe
- [ ] Screenshots are safe for public distribution
- [ ] Links point only to public resources

## Readability

- [ ] The README makes sense to a new visitor with no private context
- [ ] Each document can stand on its own without internal repo knowledge
- [ ] Claims are framed carefully and do not overstate what AI can do

## Final Review Question

If this file were posted publicly under Paras's name today, would it be safe, understandable, and professionally comfortable to stand behind? If not, revise before publishing.

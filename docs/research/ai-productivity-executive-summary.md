# Executive summary — AI productivity for data teams

## Bottom line
Top data teams are getting value from AI by **grounding it in semantic models/metric layers, embedding it in native data workflows, and keeping execution gated**.

## What is already table stakes
- semantic layer / semantic model for trusted metrics
- AI copilots in SQL, notebooks, and BI/modeling tools
- metadata quality as part of AI quality
- permission-aware context and read-only defaults
- human review for executable changes
- local measurement of workflow impact

## What is emerging edge
- agent mode that can fix/update across cells or files
- conversational analytics APIs embedded into tools/apps
- scaled AI drafting of docs/comments/tests/optimizations
- broader self-serve analytics chat beyond analysts

## Highest-value practices to adopt first
1. Curate one governed KPI domain.
2. Clean up metadata and descriptions for that domain.
3. Use AI in analyst/engineer workflows first: SQL drafting, debugging, optimization, documentation.
4. Launch business-user Q&A only on curated semantic assets.
5. Measure time-to-answer and review-accepted rates.

## Biggest risks
- wrong answers from weak semantic prep
- KPI drift from multiple definitions
- permission/privacy mistakes in context sharing
- over-broad rollout that destroys trust
- chasing prompt tricks instead of fixing metadata/model quality

## Recommendation for a data team
- **This week:** pick 1-2 domains, define trusted assets, clean metadata, choose first AI surfaces, baseline metrics.
- **This month:** publish a lightweight policy, create reusable prompt/review workflows, run one curated pilot.
- **Next 90 days:** expand carefully, add agent-like workflows only with review gates, and build a failure library + scorecard.

## One-sentence rule
**Do not ship AI on top of raw warehouse sprawl; ship it on top of curated business definitions.**

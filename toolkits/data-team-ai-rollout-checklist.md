# Data Team AI Rollout Checklist

Use this checklist to move from scattered AI experiments to a real team operating model.

## 1. Pick the Right First Use Case

- [ ] Choose a recurring workflow, not a novelty demo
- [ ] Start with a problem the team already knows how to validate
- [ ] Prefer analysis acceleration over fully autonomous decision-making
- [ ] Define what success looks like in one sentence

## 2. Set Safety Boundaries

- [ ] Decide which data can be used in AI-assisted workflows
- [ ] Establish red lines for PII, secrets, and customer data
- [ ] Require sampled, synthetic, or approved environments where needed
- [ ] Document who is accountable for final review

## 3. Standardize the Workflow

- [ ] Define a preferred prompting pattern
- [ ] Add a clarification step before building
- [ ] Add a validation or tie-out step before sharing results
- [ ] Capture corrections so the workflow improves over time

## 4. Create Shared Team Habits

- [ ] Share one or two reference workflows across the team
- [ ] Review wins and misses in team meetings
- [ ] Normalize showing failed attempts and corrected outputs
- [ ] Keep examples in a shared repo or workspace

## 5. Measure What Matters

- [ ] Track time saved on recurring work
- [ ] Track increase in output volume or analysis coverage
- [ ] Track error classes and review findings
- [ ] Track trust: are stakeholders willing to use the outputs

## 6. Train Managers, Not Just ICs

- [ ] Managers know where AI should be encouraged
- [ ] Managers know where human review is mandatory
- [ ] Managers can evaluate whether output quality is improving
- [ ] Managers can coach better workflow design, not just better prompts

## 7. Avoid the Common Traps

- [ ] Do not declare success based on one flashy demo
- [ ] Do not roll out company-wide before the workflow is reliable
- [ ] Do not treat AI usage as a substitute for metric literacy
- [ ] Do not leave validation expectations implicit

## Suggested First 30 Days

Week 1:
- pick one workflow
- define boundaries
- create the first reference example

Week 2:
- run the workflow repeatedly on known work
- document corrections
- refine prompts and review steps

Week 3:
- share with a small group
- compare outputs with baseline work
- tighten the checklist

Week 4:
- decide whether to scale, revise, or stop
- publish the preferred team pattern
- train managers on review expectations
